CKEditor 5 — Fullscreen demo

Use the toolbar Fullscreen button or press Esc to exit while fullscreen. Default behavior mounts on document.body with 100vw/100vh (and 100dvh where supported) so the editor covers the entire browser. Section B uses inPlace: true for a modal/focus-trap–friendly mode inside a transformed box.

A. Normal page (default — full viewport)

fullscreen: {} (or omit). Layer is on body and sized to the viewport.

Fullscreen: off

B. Transformed “dialog” with inPlace: true

This box uses transform: translateZ(0). With inPlace: true, the fullscreen layer stays inside the subtree (focus-trap friendly); sizing uses 100% of that containing block instead of the whole browser.

Fullscreen: off

C. Same dialog shell, default fullscreen (mounts on body)

No inPlace: editor moves to document.body and fills the full viewport even though the editor lives in this box initially. In a real Ant Design Modal, consider focusLock={false} (v5) while fullscreen if focus is trapped.

Fullscreen: off
